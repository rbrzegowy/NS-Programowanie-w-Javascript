      setTimeout i setInterval

      • setTimeout uruchamia callback raz po podanym czasie.
      • setInterval uruchamia callback cyklicznie.
      • Pamiętaj o clearInterval lub clearTimeout, gdy timer nie jest już potrzebny.
      setTimeout(() => console.log('raz'), 1000)
      const id = setInterval(() => console.log('tick'), 1000)

          Promise i fetch

          • Promise to obiekt reprezentujący operację asynchroniczną, która może zakończyć się powodzeniem lub niepowodzeniem.
          • fetch zwraca Promise z odpowiedzią.
          • Łańcuch then przetwarza dane krok po kroku.
          • catch i finally odpowiadają za obsługę błędów i cleanup.
          fetch(url).then(r => r.json())
          .then(data => ...).catch(err => ...).finally(() => ...)

              Łączenie Promise: all, allSettled, race, any

              • Promise.all - wszystko albo błąd.
              • Promise.allSettled - komplet wyników, także odrzucone.
              • Promise.race/Promise.any - wygrywa najszybszy (race - jakiekolwiek, any - pierwszy pozytywny).
              Promise.all([p1, p2, p3])
              Promise.allSettled([p1, p2, p3])

              Event Loop: mikro- i makrozadania

              • Promise.then i queueMicrotask to mikrotaski.
              • setTimeout to makrotask.
              • Mikrotaski są opróżniane przed kolejnym makrotaskiem.
              setTimeout(() => log('timeout'), 0)
              Promise.resolve().then(() => log('microtask'))

                Anulowanie operacji: AbortController

                • Przy długich requestach warto mieć możliwość anulowania.
                • AbortController działa m.in. z fetch.
                • To ważne przy przełączaniu widoków lub odmontowaniu komponentu.
                const controller = new AbortController()
                fetch(url, { signal: controller.signal })

                  Debugowanie async, fetch i XHR

                  • DevTools użyj zakładki Network i filtrów Fetch/XHR - możesz sprawdzić status, czas i payload.
                  • W Sources ustaw breakpoints w callbackach, then oraz funkcjach async.
                  • Włącz Preserve log i Disable cache, aby nie gubić kontekstu między przeładowaniami.
