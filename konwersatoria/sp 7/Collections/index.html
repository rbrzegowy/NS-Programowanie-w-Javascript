Kolekcje: Map, Set, WeakMap, WeakSet

1

Map

  • Para klucz-wartość z kluczami dowolnego typu.
  • Czyli kluczem może być dowolny obiekt, funkcja lub symbol.
  • API: set, get, has, delete, size.
const rates = new Map(); rates.set('Elden Ring', 9);

Wynik

2

Set

  • Zbiór unikalnych wartości.
  • Duplikaty są automatycznie ignorowane.
  • Set ma metody do pracy na zbiorach - m.in. .difference, .intersection, .union, .symmetricDifference, .isDisjointFrom, .isSubsetOf, .isSupersetOf
  • Więcej:https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Set#set_composition
const values = new Set([1, 1, 2]); // {1, 2}

Wynik

3

WeakMap

  • Kluczem może być tylko element "garbage-collectable"
  • Gdy referencja do obiektu znika, wpis może być usunięty przez GC z WeakMap
const cache = new WeakMap(); cache.set(userObj, data);

Wynik

4

WeakSet

  • Przechowuje tylko obiekty.
  • Nie jest iterowalny ani enumerowalny.
const visited = new WeakSet(); visited.add(nodeObj);

Wynik
